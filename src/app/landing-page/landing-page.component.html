1. Bootstrap a new application (ng new new-app --routing --style="scss" 2.Open the html that needs translating and find the label
<h1>helloworld<\h1>
 3.Use a custom id with a description.Define unique custom ids like so <h1 i18n="@@appTitle">helloworld<\h1> 4.Create a locale folder to store the xlf definition files 
(mkdir src/locale)
 5.Generate the default xlf file and place in locale folder 
(ng xi18n --output-path="locale")
6.Generate the language version xlf file and place in locale folder this is Spanish
(ng xi18n --output-path="locale" --out-file="messages.es.xlf")
 7.To build or serve in for a specific configuration/languague/locale run (ng serve -c=fr/es/it) 8. The app will now reflect the translations in the specified language, notice the LOCALIZATION tab, the dates using Angular i18n native pipes will render the date and the

Download the xliffmerge library 
npm install ngx-i18nsupport --save
 Create an xliffmerge.json file () Update package.json with the npm command to take care of the translatio and the merge in one signle command.
"extract-i18n": "ng xi18n --output-path='locale' && xliffmerge --profile xliffmerge.json fr en"
